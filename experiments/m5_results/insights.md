# M5-B Overfitting Insight: Distance-Aware Aggregation Late-Stage Accuracy Drop

## Why did accuracy drop from 85.90% (R4) to 79.69% (R10)?

The Distance-Aware strategy achieves rapid early convergence by assigning higher aggregation weights to clients whose label distributions are closest to the global reference (lowest KL divergence). In the first 4 rounds, this effectively prioritizes the most representative client, allowing the lightweight head (131K parameters) to quickly learn discriminative features from the frozen ResNet18 backbone — reaching 85.90% accuracy. However, the reference distribution itself is updated each round using the same KL-weighted average, creating a feedback loop: the reference drifts toward the dominant client's distribution, causing the weighting scheme to oscillate as different clients alternately appear "closer" or "farther" from the shifting reference. With only 1 local epoch per round, the head lacks sufficient training to compensate for these oscillating parameter contributions, and the aggregated model begins to degrade. This is compounded by the head's small capacity (two linear layers with 256 hidden units), which memorizes local noise quickly but generalizes poorly when the aggregation weights are unstable. Potential mitigations include exponential moving average (EMA) smoothing of the reference distribution, capping the maximum weight any single client can receive, increasing local epochs to stabilize per-round updates, or adding a learning rate decay schedule to reduce the magnitude of late-round parameter shifts.
